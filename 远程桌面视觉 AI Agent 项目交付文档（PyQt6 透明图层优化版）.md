# 远程桌面视觉 AI Agent 项目交付文档（PyQt6 透明图层优化版）

适用场景：向日葵 / Google 远程桌面主控端视觉识别 + 不间断自动执行 + PyQt6 透明图层交互 + 多任务管理 + 大模型工作流审批与调试

运行环境：Windows 主控电脑

架构定位：OS 级视觉自动化中枢，基于 PyQt6 透明图层构建控制台，彻底放弃 Web 页面（浏览器），支持多任务配置、大模型工作流审批，具备完整调试监测能力，无 Web 依赖、不卡顿、坐标零误差、支持 7×24h 无人值守

# 一、项目功能规格说明书（FS）（PyQt6 透明图层版）

## 1. 项目概述（优化）

本系统采用“解耦化的 OS 级 Agent 控制台”架构，彻底放弃 Web 页面（浏览器）与 Streamlit，基于 PyQt6/PySide6 构建强交互“透明图层控制台”，让 Agent 直接在真实屏幕（远程窗口）上实现视觉增强与交互。系统支持多任务自定义配置（不同任务对应不同操作内容），具备大模型工作流自动生成、人工 Approval 机制，配套前期实施检测与调试方案，实现“任务配置→透明图层同步→工作流生成→人工审批→自动执行→检测调试→页面清空→下一页循环”的完整闭环，兼顾自动化效率与人工可控性，彻底解决 Streamlit 刷新机制带来的实时控制阻碍。

## 2. 核心目标（优化）

彻底摒弃 Streamlit，聚焦 PyQt6 透明图层核心架构，核心目标如下：

- 基于 PyQt6/PySide6 构建透明图层控制台，直接叠加在远程桌面/向日葵控制界面，实现画面同步、坐标 1:1 对齐，无闪烁、无偏移、零延迟

- 支持多任务自定义配置、保存、切换，每个任务独立配置识别规则、操作目标、工作流要求，所有配置操作均在透明图层控制台完成

- 大模型可依据单个任务要求，自动设计完整工作流（含识别步骤、操作顺序、判断条件），工作流需经人工 Approval 后才可执行，审批操作在透明图层完成

- 实现工作流自动实施，配套前期检测、调试方案，确保执行过程正常，可快速定位并解决偏差、误操作等问题，调试功能集成在透明图层控制台

- 单个任务执行完成后，自动清空当前远程页面痕迹，无缝切换至下一个页面，继续执行下一个任务（支持任务队列），切换状态实时在透明图层显示

- 验证 PyQt6/PySide6 作为核心工具的优势，确保透明图层具备鼠标穿透、置顶显示、低资源占用、毫秒级刷新的特点，适配远程桌面实时控制需求

## 3. 功能模块清单（优化，移除 Streamlit 相关，强化 PyQt6 透明图层）

在原有核心模块基础上，移除 Streamlit 相关模块，新增 PyQt6 透明图层控制台模块，优化原有模块适配新架构，所有模块无缝联动，职责明确：

### 3.1 新增模块：3.8 PyQt6 透明图层控制台模块（核心）

- 透明图层渲染：基于 PyQt6/PySide6 构建全屏、置顶、无边框、鼠标穿透的透明窗口，叠加在远程桌面/向日葵控制界面之上，不影响远程操作

- 画面同步与增强：实时同步远程窗口画面，在透明图层上绘制识别框、目标点、操作提示、任务状态，实现 AR 视觉增强效果

- 多任务管理：集成任务创建、编辑、保存、删除、切换功能，所有操作均在透明图层控制台完成，每个任务关联独立的识别规则、操作目标、工作流模板

- 任务队列配置：支持在透明图层设置任务执行顺序，实现“完成一个、自动执行下一个”的循环逻辑，队列状态实时显示

- 工作流 Approval 界面：在透明图层上显示大模型生成的工作流详情，提供“审批通过”“驳回修改”“取消”交互按钮，支持人工确认后触发执行

- 调试操作入口：集成调试工具入口，支持单步执行、断点调试、坐标校准等操作，调试过程实时在透明图层显示反馈

- 日志实时展示：同步显示执行日志、调试日志、偏差日志，支持筛选查看，无需切换其他窗口

### 3.2 新增模块：3.9 实施检测与调试模块

- 执行过程监测：实时采集工作流每一步的执行状态（识别结果、操作动作、坐标准确性、页面反馈），数据同步推送至透明图层控制台

- 偏差检测：对比“大模型预期执行结果”与“实际执行结果”，自动识别坐标偏移、误操作、执行超时等问题，在透明图层弹出偏差提示

- 调试工具：提供单步执行、断点调试、坐标校准功能，支持开发/使用人员手动验证每一步操作的准确性，调试操作在透明图层完成

- 日志追溯：详细记录调试过程中的所有数据（识别帧、动作指令、偏差值），便于定位问题、优化工作流，日志可在透明图层导出

- 参数调节：支持在透明图层动态调整 AI 识别频率、执行延迟、坐标偏移校准值，快速适配不同任务场景，调节后实时生效

### 3.3 优化模块：3.4 AI 视觉理解与决策模块（适配透明图层）

### 3.4 优化模块：3.5 任务状态机与不间断执行模块（适配透明图层）

- 高性能低占用：相比 Streamlit 占用大量前端资源，PyQt6 运行轻量化，CPU、内存占用低，支持 7×24h 无人值守稳定运行，无内存溢出风险

## 模块间数据流（优化，移除 Streamlit 相关，适配 PyQt6 架构）

在原有数据流基础上，移除 Streamlit 相关联动，新增 PyQt6 透明图层控制台与各模块的联动逻辑，确保数据流转顺畅：

1. PyQt6 透明图层控制台 → 窗口锁定模块：获取远程窗口坐标，确保透明图层与远程窗口坐标 1:1 对齐，同步窗口异动信息

2. PyQt6 透明图层控制台 → 任务状态机模块：下发任务配置、任务队列顺序、工作流审批结果、调试指令

3. AI 决策模块 → PyQt6 透明图层控制台：推送大模型生成的工作流、识别结果，供人工审批与视觉展示

4. 任务状态机模块 → PyQt6 透明图层控制台：发送任务执行状态、任务完成信号，触发页面清空操作与状态提示

5. 视觉采集模块 → PyQt6 透明图层控制台：推送实时画面，实现远程画面同步与 AR 视觉增强绘制

6. 检测调试模块 → PyQt6 透明图层控制台：推送执行监测数据、偏差信息、调试日志，供实时查看与操作

7. 检测调试模块 → 所有模块：采集执行数据、识别偏差，推送调试指令；同步调试日志到 PyQt6 透明图层控制台

8. 检测调试模块 → AI 决策模块：推送偏差数据，辅助大模型优化工作流

9. PyQt6 透明图层控制台 → 日志与监控模块：获取执行日志、调试日志、资源监控数据，实时展示并支持导出

# 三、接口定义文档（模块间调用规范）（PyQt6 透明图层版）

说明：全部为进程内函数/类接口，无网络接口；统一返回格式：{ success: bool, data: any, msg: str }；所有接口线程安全，支持异步调用；移除 Streamlit 相关接口，新增 PyQt6 透明图层控制台模块接口，优化原有接口适配新架构，不改变原有接口使用方式。

## 1. 新增模块接口：3.8 PyQt6 透明图层控制台模块接口

### 1.1 PyQtOverlay.init_overlay(window_rect)

- 功能：初始化 PyQt6 透明图层窗口，贴合远程窗口坐标，设置透明、置顶、鼠标穿透属性

- 入参：window_rect: 远程窗口坐标结构体（left, top, width, height）

- 返回：{ success: bool, msg: "透明图层初始化成功/失败", data: { overlay_handle: 窗口句柄 } }

### 1.2 PyQtOverlay.sync_remote_frame(frame)

- 功能：同步远程窗口画面到 PyQt6 透明图层，实现画面实时同步，为 AR 绘制提供基础

- 入参：frame: 图像对象（视觉采集模块获取的远程窗口截图）

- 返回：{ success: bool, msg: "画面同步成功/失败", data: { sync_status: "同步/异常" } }

### 1.3 PyQtOverlay.draw_ar_elements(elements)

- 功能：在透明图层上绘制 AR 元素（识别框、目标点、提示文字、动画），实现视觉增强

- 入参：elements: 元素列表（含元素类型、坐标、尺寸、颜色、提示文本等）

- 返回：{ success: bool, msg: "AR 元素绘制成功/失败" }

### 1.4 TaskConfig.create_task(task_info)

- 功能：在透明图层控制台创建新任务，配置任务相关信息（操作内容、识别规则、目标要求）

- 入参：
        {
  "task_id": "task_001",
  "task_name": "外卖接单任务",
  "target_content": "识别外卖新订单、点击接单按钮、填写备注",
  "recognition_rules": {}, // 该任务专属的识别规则
  "priority": 1 // 任务优先级，用于队列排序
}

- 返回：任务创建结果，包含任务完整信息，同步更新透明图层任务列表

### 1.5 TaskConfig.set_task_queue(task_ids)

- 功能：在透明图层控制台设置任务执行队列，定义任务执行顺序

- 入参：task_ids: list（任务ID列表，按执行顺序排列）

- 返回：队列设置结果，透明图层实时显示队列顺序

### 1.6 WorkflowApproval.submit_approval(workflow_id, approval_status)

- 功能：在透明图层控制台提交工作流审批结果（通过/驳回/取消）

- 入参：
       {
  "workflow_id": "workflow_001",
  "approval_status": "pass/reject/cancel",
  "edit_content": {} // 驳回时的人工编辑内容（可选）
}

- 返回：审批结果，若通过则触发工作流执行信号，透明图层显示审批状态

### 1.7 PyQtOverlay.clear_remote_page()

- 功能：触发远程页面清空操作，同时在透明图层显示“页面清空中”提示，清空完成后提示“准备执行下一个任务”

- 入参：无

- 返回：页面清空结果，同步更新透明图层任务状态

### 1.8 PyQtOverlay.show_debug_info(debug_info)

- 功能：在透明图层控制台显示调试信息（单步执行结果、偏差信息、校准值）

- 入参：debug_info: 调试详情（步骤、偏差、操作、校准值等）

- 返回：调试信息显示结果

### 1.9 PyQtOverlay.update_log(log_info)

- 功能：在透明图层控制台实时更新执行日志、调试日志，支持筛选查看

- 入参：log_info: 日志详情（时间、类型、内容、关联任务/工作流）

- 返回：日志更新结果

## 2. 新增模块接口：3.9 实施检测与调试模块接口（无修改，适配透明图层数据流）

### 2.1 DebugMonitor.monitor_execution(step_info)

- 功能：实时监测工作流每一步的执行情况，采集执行数据，同步推送至透明图层控制台

- 入参：
        {
  "workflow_id": "workflow_001",
  "step_id": "step_001",
  "action": "CLICK/INPUT",
  "expected_result": {}, // 大模型预期执行结果
  "actual_result": {}, // 实际执行结果
  "coords": {x:100, y:200}, // 执行坐标
  "time": "2024-05-01 10:00:00"
}

- 返回：监测结果，是否存在偏差，偏差信息同步推送至透明图层

### 2.2 DebugMonitor.detect_deviation(expected, actual)

- 功能：对比预期结果与实际结果，自动识别执行偏差（坐标偏移、误操作等），在透明图层弹出偏差提示

- 入参：expected: 预期结果结构体，actual: 实际结果结构体

- 返回：{ has_deviation: bool, deviation_info: { type: "坐标偏移/误操作", value: 偏差值 } }

### 2.3 DebugTool.single_step_execution(step_id)

- 功能：通过透明图层控制台触发单步执行工作流某一步，用于调试验证，执行结果实时显示在透明图层
